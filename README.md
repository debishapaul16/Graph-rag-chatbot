# Hybrid GraphRAG Chatbot using LangChain, Neo4j, ChromaDB and Gemma

## Overview

This project presents a complete implementation of a **Hybrid Graph Retrieval-Augmented Generation (Hybrid GraphRAG)** system built using Python. The application combines semantic vector retrieval with knowledge graph retrieval to improve the quality and contextual accuracy of responses generated by a Large Language Model (LLM).

The project demonstrates the complete GraphRAG pipeline, including document preprocessing, embedding generation, vector database storage, knowledge graph construction, graph querying, hybrid retrieval, and answer generation.

The implementation is provided in a step-by-step Jupyter Notebook format with detailed explanations, making it suitable for both learning and practical implementation.

# Project Workflow

The implementation follows the complete Hybrid GraphRAG pipeline:

1. Load the input document.
2. Split the document into overlapping text chunks.
3. Generate dense vector embeddings for every chunk.
4. Store embeddings in ChromaDB.
5. Extract entities from the document.
6. Store extracted entities as nodes in Neo4j.
7. Extract relationships between entities using the LLM.
8. Store relationships in the Knowledge Graph.
9. Accept a user query.
10. Extract the primary entity from the query.
11. Retrieve related information from the Knowledge Graph.
12. Retrieve semantically relevant chunks from ChromaDB.
13. Combine graph context and vector context.
14. Generate the final prompt.
15. Produce the final answer using the LLM.

# Notebook Description

## 1. Document Loading

* Read the input document.
* Display document contents.
* Prepare the document for processing.

---

## 2. Text Splitting

* Split large documents into smaller chunks.
* Configure chunk size.
* Configure chunk overlap.
* Prepare chunks for embedding generation.

---

## 3. Embedding Generation

* Load the Sentence Transformer model.
* Generate dense embeddings for every chunk.
* Prepare vectors for storage.

---

## 4. ChromaDB Storage

* Create a persistent ChromaDB client.
* Create or access a collection.
* Store embeddings.
* Verify stored vectors.

---

## 5. Semantic Vector Search

* Convert the user query into an embedding.
* Search the vector database.
* Retrieve the most relevant document chunks.

---

## 6. Entity Extraction

* Read the source document.
* Extract entities using regular expressions.
* Remove duplicate entities.
* Display extracted entities.

---

## 7. Knowledge Graph Construction

* Connect to Neo4j.
* Store entities as graph nodes.
* Build the initial Knowledge Graph.

---

## 8. Relationship Extraction

* Use the LLM to identify relationships.
* Convert the response into structured JSON.
* Store graph relationships in Neo4j.

---

## 9. Knowledge Graph Search

* Execute Cypher queries.
* Retrieve related entities.
* Display graph relationships.

---

## 10. Hybrid Search

* Extract the primary entity from the user query.
* Perform Knowledge Graph retrieval.
* Perform Vector Database retrieval.
* Merge both retrieval contexts.

---

## 11. Answer Generation

* Construct the final prompt.
* Provide both retrieval contexts to the LLM.
* Generate a context-aware answer.

# Installation

## Clone the Repository

```bash
git clone https://github.com/<your-username>/Hybrid-GraphRAG.git

cd Hybrid-GraphRAG
```

---

## Create a Virtual Environment

### Windows

```bash
python -m venv venv

venv\Scripts\activate
```

### Linux / macOS

```bash
python3 -m venv venv

source venv/bin/activate
```

---

## Install Dependencies

```bash
pip install -r requirements.txt
```

---

# Install Ollama

Install Ollama from the official website.

After installation, download the Gemma model:

```bash
ollama pull gemma3
```

Run the model:

```bash
ollama run gemma3
```

---

# Configure Neo4j

1. Install Neo4j Desktop.
2. Create a local database.
3. Start the database.
4. Update the following values inside the notebook:

* Neo4j URI
* Username
* Password
